## 2025-05-19 12:00:00 - [Android IME + Compose + ML Kit Data Flow]
Learning: The custom OCR keyboard operates entirely within an `InputMethodService` (via `LifecycleInputMethodService`). It presents UI using Jetpack Compose (via `ComposeView` returned by `onCreateInputView`), passing user intentions to a ViewModel. Crucially, the captured image never leaves the device. The ViewModel delegates to an `OcrRepositoryImpl` which executes ML Kit text recognition natively. Extracted text is then emitted via a Flow back to the service, which acts as the ultimate sink, committing the text directly to the target application's `InputConnection`.
Action: Always document the unidirectional data flow from the Compose UI layer to the `InputMethodService` sink when explaining feature architecture in this repo.
